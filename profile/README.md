# CoffeeCommunity ☕️

CoffeeCommunity는 커피를 좋아하는 사람들이  
자유롭게 소통하고 경험을 공유할 수 있는  
**커뮤니티 기반 서비스와 웹 플랫폼**을 개발하는 조직입니다.

단순한 정보 나열이나 일회성 콘텐츠에 그치지 않고,  
사용자 간의 경험, 취향, 이야기가 자연스럽게 쌓이고  
지속적으로 확장될 수 있는 커뮤니티 환경을 만드는 것을 목표로 합니다.

우리는 기술 자체보다 **사용자 경험과 실제 사용성**을 우선하며,  
작은 기능 하나라도 “왜 필요한가”를 고민하며 개발합니다.

---

## 🚀 What We Do

CoffeeCommunity는 커피라는 공통 관심사를 중심으로  
다양한 사용자 참여형 기능을 제공하는 웹 서비스를 만들고 있습니다.

- 커피 관련 정보 공유 커뮤니티  
  (원두, 로스팅, 브루잉, 카페 리뷰 등)

- 사용자 참여형 콘텐츠 시스템  
  (게시글, 댓글, 리뷰, 공감/반응)

- 인증 기반 사용자 시스템  
  (회원 가입, 로그인, 권한 관리)

- 확장 가능한 서비스 구조 설계  
  (기능 추가와 유지보수를 고려한 아키텍처)

우리는 단순한 CRUD 구현을 넘어서,  
실제 서비스 환경을 가정한 구조와 흐름을 설계하고 구현합니다.

---

## 🛠 Tech Stack

### Frontend
- React
- TypeScript
- Tailwind CSS

컴포넌트 기반 구조를 바탕으로  
재사용성과 유지보수를 고려한 UI/UX를 설계합니다.

---

### Backend
- Spring Boot
- JPA / Hibernate
- JWT 기반 인증

도메인 중심 설계를 통해  
비즈니스 로직을 명확하게 분리하고,  
안정적인 인증/인가 구조를 구현합니다.

---

### Infra & Tools
- Docker
- GitHub Actions
- REST API
- GitHub Issues / Pull Requests 기반 협업

개발부터 배포까지의 흐름을 고려하며  
자동화와 협업 효율을 중요하게 생각합니다.

---

## 📦 Main Projects

- `coffeecommunity-backend`  
  → 커뮤니티 도메인 로직, 인증 시스템, API 서버 구현

- `coffeecommunity-frontend`  
  → 사용자 인터페이스 및 사용자 경험 중심의 웹 클라이언트

- `coffeecommunity-infra`  
  → 배포 환경 설정, CI/CD 구성, 인프라 관련 관리

각 프로젝트는 역할과 책임이 명확히 분리되어 있으며,  
실제 서비스 운영을 가정한 구조로 관리됩니다.

---

## 👥 How We Work

CoffeeCommunity는 협업과 코드 품질을 중요하게 생각합니다.

- 기능 단위로 명확한 역할 분리
- Issue 기반 작업 관리
- Pull Request를 통한 코드 리뷰
- 가독성과 유지보수를 고려한 코드 스타일
- 공통 컨벤션과 문서화를 통한 협업 효율 향상

단순히 “작동하는 코드”가 아니라,  
**함께 읽고, 함께 개선할 수 있는 코드**를 지향합니다.

## 🌱 Vision

커피라는 일상적인 주제를 통해  
사람들이 자연스럽게 모이고,  
자유롭게 이야기하며,  
지속적으로 성장할 수 있는 커뮤니티를 만드는 것이 목표입니다.

작은 서비스라도 완성도 있게,  
사용자에게 실제로 가치 있는 경험을 제공하는  
**지속 가능한 커뮤니티 플랫폼**을 만들어갑니다.
